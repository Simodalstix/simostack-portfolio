Database Migration PostgresSQL to Amazon RDS

This project focused on a realistic migration scenario: taking a local PostgreSQL database and moving it to the cloud using AWS RDS. I designed the architecture with Terraform, including a VPC with public and private subnets, a Bastion host accessed via SSM Session Manager, and Secrets Manager for secure credential handling — avoiding any exposed ports or direct SSH.

The database was staged through S3 and restored using psql, closely mimicking how a secure cloud transition might be handled in production. It gave me practical insight into infrastructure planning, data transfer, and how to combine security best practices with infrastructure-as-code.

Raspberry Pi Backend with CI/CD & NGINX Proxy

This project explores a cost-effective backend deployment using a Raspberry Pi. I containerized a Node.js + PostgreSQL API with Docker Compose and used GitHub Actions to auto-deploy on push via SSH. To expose the API securely, I configured NGINX Proxy Manager in Docker, routing traffic from a custom subdomain (api.simostack.com) via DuckDNS with SSL from Let’s Encrypt.

Persistent volumes handle data durability. The result is a cloud-style deployment — self-hosted for zero cost — and a great way to build practical skills in Docker, CI/CD, and network routing.
